## What's NOT Implemented (Requires Language Features)

### 1. Array Functional Methods
**Requires:** First-class functions, closures, lambda expressions

**Missing:**
```wyn
// These CANNOT be implemented without closures
arr.map(fn)           // Transform each element
arr.filter(fn)        // Filter elements
arr.reduce(fn, init)  // Reduce to single value
arr.find(fn)          // Find first match
arr.any(fn)           // Check if any match
arr.all(fn)           // Check if all match
```

**Why:** Wyn has no way to pass functions as arguments. The language needs:
- Function type syntax: `fn(int) -> int`
- Lambda expressions: `|x| x * 2`
- Closure support

**Workaround:** Use while loops:
```wyn
// Instead of: arr.map(|x| x * 2)
var result = [];
var i = 0;
while i < arr.len() {
    result.push(arr[i] * 2);
    i = i + 1;
}
```

### 2. Advanced Networking
**Requires:** Async/await, event loop, threading

**Missing:**
- `Net::accept()` - Accept connections (would block forever)
- Async I/O
- HTTP parsing library
- WebSocket support
- TLS/SSL

**Why:** Current implementation is synchronous blocking I/O. Real servers need:
- Non-blocking sockets
- Event loop (epoll/kqueue)
- Thread pool or async runtime

**Workaround:** Basic TCP is sufficient for simple clients and testing.

### 3. Regex
**Requires:** Complex parser or external library

**Missing:**
- `Regex::new(pattern)`
- `Regex::match(text)`
- `Regex::replace(text, replacement)`

**Why:** Would need full regex engine (thousands of lines of code) or FFI to PCRE library.

**Workaround:** Use `.contains()`, `.starts_with()`, `.ends_with()`, `.split()` for simple patterns.
